Select Tagged Objects

This page displays all the Tagged Repository Objects that were found for a given list of filter criteria (like e.g. tags)

Page Layout

The page basically consists of Table with the following columns:

  • Tag
    The name of the assigned tag of the object
  • Object
    The name of the object/component
  • Parent Tag (if necessary)
    The name of the parent tag of the assigned parent object
  • Parent Object (if necessary)
    The name of the assigned parent object
  • Issues
    Potential issues that prevent the removal of the tag

Keyboard Shortcuts

Shortcut Function
Ctrl+F Jump to the filter field if focus is on the objects table

Why can a tag not be removed?

There a persistent and transient issues why a tag may not be removed from an object.

Let's assume we have the following tag hierarchy

  • Root
    • Sub

Now we have an object OBJ1 assigned to tag Root and an object OBJ2 assigned to tag Sub with parent object OBJ1
If we only select objects with the tag Root, the tagged object OBJ1 will get a persistent issue that there is still a child object usage.

On the other hand, if both the tags Root and Sub were selected we would get both tagged objects in the result table.
If the object OBJ2 with the tag Sub would now be deselected, the object OBJ1 with tag Root would be deselected as well and the tagged object OBJ1 would get a transient issue. Once the object OBJ2 is selected again for deletion, the object OBJ1 would be deletable once again.
